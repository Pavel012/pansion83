Отчет "Аудит выданных карт идентификации клиентов"

Назначение

Имя отчета в конфигураторе «IdentificationCardsAudit».

Отчет выводит список зарегистрированных в системе карт идентификации клиентов. Карты регистрируются либо через интерфейс с замковой системой, либо по считывателю. Отчет позволяет сгруппировать выданные карты по клиенту и контролировать выданное кол-во карт по клиенту и номеру комнаты.

Отборы на форме отчета

  • Период отчета – отбирает за какой период сформировать отчет. Карты отбираются по дате и времени регистрации в программе 
  • Гостиница – задает отбор по гостинице, если поле оставить пустым, то будут выведены данные по всем гостиницам.
  • Номер – задает отбор по номеру комнаты или по группе номеров.
  • Тип номера – задает отбор по типу номера или по группе типов номеров.
  • Автор (сотрудник) – задает отбор по сотрудникам, которые регистрировали карты в системе.

Группировки

  • Автор – Сотрудник выдавший карту.
  • Номер комнаты – Номер комнаты по которому выдана карта.
  • Тип номера – Категория номера.
  • Учетная дата – Дата выдачи (регистрации) карты в системе (без времени).
  • Группа – объединяет несколько гостей, но может также состоять и из одного гостя. Каждый гость обязательно принадлежит какой – либо группе. Номер группы – это номер брони. В разрезе групп ведется учет взаиморасчетов с контрагентами.
  • Гостиница – название гостиницы, в которой размещен гость. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
  • Документ основание – ссылка на документ «Размещение» или бронь.
  • Фолио – лицевой счет прикрепленный к карте.
