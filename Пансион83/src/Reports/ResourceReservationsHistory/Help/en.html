Отчет "История бронирования ресурсов"

Назначение

Имя отчета в конфигураторе «ResourceReservationsHistory».

Отчет выводит всю бронь ресурсов за указанный период по указанным гостиницам, ресурсам и типам ресурсов.

Отборы на форме отчета

  • По виду проверки периода отчета - проверка дат брони относительно периода формирования отчета. Проверок существует насколько видов:
    • пересечение - в отчете отобразятся брони, период бронирования которых пересекается с периодом отчета;
    • начинается в периоде - в отчет попадут только те брони, дата начала которых попадает в запрашиваемый период;
    • заканчивается в периоде - в отчет попадут только те брони, дата окончания которых попадает в запрашиваемый период;
    • начинается или заканчивается в периоде - в отчет попадут только те брони, дата начала или окончания которых попадает в запрашиваемый период;
    • дата создания документа попадает в период - в отчете отобразятся брони, которые были созданы в периоде отчета.
  • По ресурсу - задает отбор по ресурсу.
  • По типу ресурса - задает отбор по типу ресурса, если поле оставить пустым, то будут выведенные данные по всем типам ресурсов гостиницы.
  • По типу брони - может быть действующей (т.е. введена, имеет статус «Бронь», но по ней еще не заехали гости) и недействующей.
  • По группе гостей.

Группировки

  • Счет (Счет–требование или Фолио) – ссылка на соответствующий документ.
  • Валюта счета – валюта документа Счет–требование или Фолио.
  • Агент – ссылка на справочник контрагентов, указывает контрагента, который получает комиссию за гостя. Может также использоваться для аналитики работы агентов или туроператоров.
  • Автор – пользователь, автор документа «Бронь ресурса».
  • Клиент – гость, ссылка на справочник «Клиенты».
  • Тип клиента – характеристика гостя, может использоваться как разделитель учета для аналитики, также используется для задания разных цен по разным типам клиентов (каждому типу клиента можно поставить в соответствие свой тариф).
  • Фирма – юр.лицо гостиницы, от имени которой оформлено размещение гостя.
  • Контактное лицо – ссылка на одноименное текстовое поле в документе «Бронь», указывает на лицо, бронировавшее данный номер от имени клиента. Обычно указывается при бронировании для контрагентов.
  • Договор – ссылка на договор контрагента, используется как разделитель учета взаиморасчетов с контрагентом.
  • Контрагент – ссылка на справочник контрагентов, указывает на организацию, юр. лицо, от имени которого было произведено бронирование (размещение).
  • Тип контрагента – характеристика контрагента, может использоваться как разделитель учета для аналитики, также используется для задания разных цен по разным типам контрагентов (каждому типу контрагентов можно поставить в соответствие свой тариф).
  • Дисконтная карта – ссылка на справочник дисконтных карт. Заполняется при поселении, если гость предъявил карту.
  • Тип скидки – разделитель учета, ссылка на справочник «Тип скидок». Установка соответствия между набором услуг и размером скидки.
  • День брони - дата начала действия брони.
  • Час брони - час начала действия брони.
  • Месяц брони - номер месяца начала действия брони.
  • Квартал брони - номер квартала начала действия брони.
  • Неделя брони - номер недели начала действия брони.
  • Год брони - год начала действия брони.
  • Группа – объединяет несколько гостей, но может также состоять и из одного гостя. Каждый гость обязательно принадлежит какой–либо группе. Номер группы – это номер брони. В разрезе групп ведется учет взаиморасчетов с контрагентами.
  • Гостиница – название гостиницы, в которой размещен гость. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
  • Направление маркетинга – ссылка на элемент справочника «Направления маркетинга». Используется для анализа, может быть использован в группировках и отборах.
  • Документ основание - документ, на основании которого создана текущая бронь ресурсов.
  • Планируемый способ оплаты – ссылка на справочник «Способы оплаты». Разделитель учета, позволяющий указать, каким образом осуществляется оплата текущего гостя.
  • Регистратор – ссылка на документ «Бронь ресурса».
  • Ресурс – ссылка на справочник «Ресурсы», указывает забронированный ресурс.
  • Статус брони – указывает текущий статус забронированного ресурса.
  • Тип ресурса – указывает на тип забронированного ресурса.
  • Источник информации – ссылка на справочник «Источник информации», может использоваться для построения аналитических отчетов.

Колонки

  • Баланс по счету – указывает на состояние взаиморасчетов клиента и гостиницы. Баланс положителен если это долг гостя, и отрицателен в случае долга гостиницы.
  • Комиссия агента – сумма вычисленной комиссии для агента.
  • Тип комиссии агента.
  • Автомобиль.
  • Время начала брони – указывает на момент, с которого начинает действовать бронь данного ресурса.
  • Время окончания брони – указывает на момент, с которого кончает действовать бронь данного ресурса.
  • Количество оставшихся дней до начала периода брони – разница между текущей датой и датой начала брони.
  • Скидка – сумма предоставленной скидки.
  • Продолжительность – длительность периода бронирования (обычно в часах).
  • Возраст счета в днях – разница между датой документа «Счет-требование» и текущей датой.
  • Человек – количество человек, которое пользуется данным ресурсом в данной группе.
  • Примечания - текстовый комментарий.
  • Сумма счета – начисленная за бронирование ресурса сумма.
  • Оплата по счету – сумма, оплаченная по данному счету требованию (фолио).
