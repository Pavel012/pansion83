Отчет "Сводка по оказанным услугам"

Назначение

Имя отчета в конфигураторе «ServicesSales».

Отчет позволяет получить детальную распечатку услуг оказанных за период.

Отборы на форме отчета

  • По набору услуг - задает отбор по набору услуг, если поле оставить пустым, то будут выведены данные по всем услугам гостиницы.
  • По услуге - задает отбор по услуге, если поле оставить пустым, то будут выведены данные по всем услугам гостиницы.
  • По номеру комнаты.

Группировки

  • Учетная дата – дата оказания услуг.
  • Агент – ссылка на справочник контрагентов, указывает контрагента, который получает комиссию за гостя. Может также использоваться для аналитики работы агентов или туроператоров.
  • Автор – пользователь, автор документа «Размещение».
  • Клиент – гость, ссылка на справочник «Клиенты».
  • Возраст гостя.
  • Гражданство гостя.
  • Город гостя.
  • Регион гостя.
  • Тип клиента – характеристика гостя, может использоваться как разделитель учета для аналитики, также используется для задания разных цен по разным типам клиентов (каждому типу клиента можно поставить в соответствие свой тариф).
  • Фирма – юр.лицо гостиницы, от имени которой оформлено размещение гостя.
  • Договор – ссылка на договор контрагента, используется как разделитель учета взаиморасчетов с контрагентом.
  • Контрагент – ссылка на справочник контрагентов, указывает на организацию, юр. лицо, от имени которого было произведено бронирование (размещение).
  • Фолио – ссылка на Лицевой счет.
  • Группа – объединяет несколько гостей, но может также состоять и из одного гостя. Каждый гость обязательно принадлежит какой–либо группе. Номер группы – это номер брони. В разрезе групп ведется учет взаиморасчетов с контрагентами.
  • Гостиница – название гостиницы, в которой размещен гость. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
  • Это доход от продажи ресурсов - принимает значение «Да», если услуга принадлежит ресурсу (в карточке услуги должен быть проставлен соответствующий признак). В противном случае - «Нет».
  • Это доход от продажи номеров - принимает значение «Да», если это доход от продажи номеров (в карточке услуги должен быть проставлен соответствующий признак). В противном случае - «Нет».
  • Сторно.
  • Направление маркетинга – ссылка на элемент справочника «Направления маркетинга». Используется для анализа, может быть использован в группировках и отборах.
  • Документ основание – ссылка на документ, являющийся основанием для начисления услуги.
  • Цена – цена начисленной услуги.
  • Регистратор – ссылка на документ «Начисление».
  • Отчетная валюта – валюта, указанная в качестве отчетной для данной гостиницы.
  • Ресурс – ссылка на справочник «Ресурсы», указывает ресурс, использованный при оказании услуги.
  • Номер – указывает номер комнаты, в которой размещен клиент.
  • Тариф – показывает, по какому тарифу поселили гостя, или забронировали номер. Тариф определяет цену номера и набор начисляемых услуг.
  • Тип тарифа – разделитель учета, в разрезе типа тарифа можно строить аналитические отчеты.
  • Тип номера – ссылка на справочник «Типы номеров», указывает на тип номера, в котором размещен гость (планируется размещение).
  • Услуга – название начисленной услуги.
  • Источник информации.
  • Цель поездки – ссылка на справочник «Цели визита», может использоваться для построения аналитических отчетов.
  • Ставка НДС – размер ставки НДС.

Колонки

  • Комиссия – сумма вычисленной комиссии для агента.
  • Комиссия без НДС – та же сумма без НДС.
  • Сумма скидки – сумма предоставленной скидки.
  • Сумма скидки без НДС – та же сумма без НДС.
  • Доп. Мест – количество дополнитеотных мест в номере.
  • Мест – количество мест в номере.
  • Количество мест в номере.
  • Количество человек – количество гостей, проживающих в номере.
  • Количество человек в номере.
  • Номеров – количество номеров.
  • Количество – количество начисленной услуги.
  • Сумма – произведение цены на количество.
  • Сумма без НДС – та же сумма за вычетом НДС.
  • Момент времени.
