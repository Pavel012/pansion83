Отчет "Оперативный баланс взаиморасчетов с контрагентами"

Назначение

Имя отчета в конфигураторе «CustomerAccountsOperational».

Отчет позволяет распечатать оперативный баланс взаиморасчетов с контрагентами.

Оперативный баланс учитывает услуги, которые еще не закрыты актами и планируемые услуги по брони. В общем случае оперативный баланс рассчитывается по формуле: 

Оперативный баланс = (Сумма всех начисленных услуг – Сумма всех платежей) + Планируемое оказание услуг по действующей брони
Сумму всех начисленных услуг можно разделить на 2 части – те услуги, что уже оформлены актами и переданы в бухгалтерию и те услуги, что уже начислены, но актами еще не оформлены и в бухгалтерию еще не переданы, которые обычно относятся к еще проживающим гостям. В отчете есть параметр «Бухгалтерский баланс», который равен сумме всех актов минус сумма всех платежей. Значение «Бух. баланса» можно сверить с отчетом «Баланс взаиморасчетов с контрагентами» и с балансом по счету 62 в бухгалтерии. С учетом этого параметра формулу оперативного баланса можно выразить следующим образом: 
Оперативный баланс = Бухгалтерский баланс + Начисленные услуги по проживающим гостям (не оформленные актами) + Планируемое оказание услуг по действующей брони
Данный отчет можно использовать для прогноза взаиморасчетов с контрагентом. Например, если во взаиморасчетах принята схема когда контрагент вносит предоплату и затем бронирует номера в пределах этой предоплаты, то сумма «Оперативного баланса» как раз отражает на какую сумму можно еще делать бронь.

Если параметр «Период по» на форме отчета не заполнен то учитываются все бронирования. Если же параметр «Период по» заполнен, то в балансе будут учтены не все услуги, а только те услуги, которые оказаны или планируется оказать до указанной даты.

Отборы на форме отчета

  • По гостинице - задает отбор по гостинице, если поле оставить пустым, то будут выведены данные по всем гостиницам.
  • По фирме - если указано, то в отчет будут выведены только данные указанной фирмы. Если реквизит пустой, то будут выведены данные по всем фирмам.
  • По валюте взаиморасчетов.
  • По основным разрезам учета взаиморасчетов – по контрагенту, договору и группе гостей.

Группировки

  • Фирма – юр.лицо гостиницы (организация) по которой проводили начисление. Используется как разделитель учета при многофирменном учете.
  • Договор – договор контрагента, ссылка на справочник, используется для учета по договорам. Если учет ведется без договоров, то поле и колонка остаются пустые.
  • Валюта – валюта взаиморасчетов. Устанавливается в договоре или у контрагента. В этой валюте проводятся документы – акты и платежи.
  • Контрагент – ссылка на справочник контрагентов.
  • Группа – номер группы гостей, ссылка на справочник групп. Группа назначается автоматически при бронировании. Номер группы соответствует номеру брони. В отчет можно также добавлять отдельные колонки с реквизитами группы, такими как описание группы, клиент, дата заезда и выезда, продолжительность проживания.
  • Гостиница – название гостиницы, в которой размещен гость. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.

Колонки

  • Бухгалтерский баланс на конец периода («Бух. баланс») – баланс по регистру взаиморасчетов с контрагентами, который учитывает Акты и Платежи и равен сумме всех услуг оформленных актами минус сумма всех платежей.
  • Сумма оказания («Оказано») – сумма начисленных услуг по проживающим гостям, но которые еще не были оформлены актами.
  • Планируемая сумма оказания по брони («План. по брони») – сумма всех услуг по действующей брони.
  • Оперативный баланс («Баланс») – значение баланса с учетом всех будущих услуг. Рассчитывается по формуле:
     «Баланс» = «Бух.баланс» + «Оказано» + «План по брони»
