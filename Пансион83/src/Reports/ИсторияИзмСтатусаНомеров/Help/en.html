Отчет "История изменения статусов номеров"

Назначение

Имя отчета в конфигураторе «RoomStatusChangeHistory».

Отчет позволяет распечатать аудит истории изменения статусов номеров за период.

Отборы на форме отчета

  • По статусу номера - проверка статуса номера относительно периода формирования отчета.
  • По номеру - задает отбор по номеру, если поле оставить пустым, то будут выведенные данные по всем номерам гостиницы.
  • По типу номера - задает отбор по типу номера, если поле оставить пустым, то будут выведенные данные по всем типам номеров гостиницы.
  • По сотруднику - задает отбор по сотруднику. Если в поле будет выбран конкретный сотрудник, то можно будет получить отчет со списком всех изменений статусов номеров, которые были сделаны именно им.

Группировки

  • Учетная дата – дата изменения статуса номера.
  • Учетный день – день изменения статуса номера.
  • Учетный месяц – номер месяца изменения статуса номера.
  • Учетный квартал – номер квартала изменения статуса номера.
  • Учетная неделя – номер недели изменения статуса номера.
  • Учетный год – год изменения статуса номера.
  • Сотрудник – пользователь, изменивший статус номера.
  • Гостиница – название гостиницы, в которой размещен гость. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
  • Номер – указание номера.
  • Статус номеров – ссылка на справочник «Статусы номеров», указывает статус номера, который обычно меняется при заезде и выселении из номера и обычно принимает значения «Занят», «Свободен», «Грязный», «Чистый».
  • Тип номера – ссылка на справочник «Типы номеров», указывает тип номера, статус которого меняется.

Колонки

  • Период - дата и время изменения статуса.
  • Примечания - если статус номера был изменен программой автоматически при заселении или выселении гостя, то примечание будет заполнено. Если примечание пустое, значит, статус был изменен вручную.
