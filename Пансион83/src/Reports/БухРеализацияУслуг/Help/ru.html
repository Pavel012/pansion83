Отчет "Бухгалтерская реализация услуг"

Назначение

Имя отчета в конфигураторе «AccountsReceivable».

Данный отчет формируется на основании проведенных актов. Источником для отчета служит регистр накопления «Реализация» (AccountsReceivable). Отчет может использоваться для сверки с бухгалтерией или для проверки выгруженных актов. Сумма этого отчета за выгружаемый период должна равняться сумме выгруженных актов. В случае когда выгрузка данных в бухгалтерию не используется, данные по реализации распечатываются и передаются в бухгалтерию на бумаге.

Отборы на форме отчета

  • Период отчета – отбирает за какой период включать акты в отчет. Внимание, отбор идет по дате акта, а не по дате начисления
  • Фирма – если указано, то в отчет будут выведены только данные по актам указанной фирмы. Если реквизит пустой, то будут выведены данные по всем фирмам.
  • Гостиница – задает отбор по гостинице, если поле оставить пустым, то будут выведены данные по всем гостиницам.
  • Контрагент – задает отбор по контрагенту или группе контрагентов, если не выбран, то данные будут выведены по всем контрагентам.
  • Договор – задает отбор по договору контрагента.
  • Группа – отбор по группе гостей.
  • Набор услуг – если поле заполнено, то в отчет попадут только данные по услугам входящим в выбранный набор.

Группировки

  • Контрагент – указывает контрагента, на которого был оформлен акт.
  • Договор – договор контрагента. Если гостиница не использует договоры в учете, то данную группировку можно исключить.
  • Группа – номер группы, на которую оформлен акт. Данная группировка используется, когда бухгалтеру нужно на каждую группу, на каждый номер брони, создавать отдельные акт и счет – фактуру.
  • Номер – номер комнаты, в которой проживал гость.
  • Клиент – гость, проживавший в номере.
  • Услуга – по какой услуге прошли начисления.
  • Фирма – группировка используется, только в случае, если в гостинице предусмотрен многофирменный учет. В случае, когда все операции ведутся от имени одного юридического лица, эту группировку лучше убрать для экономии места в отчете.
  • Гостиница – для учета в одной базе по нескольким гостиницам. Обычно группировка в отчет не включается, а на каждую гостиницу составляется отдельный отчет. Быстрый отбор по гостинице можно производить на форме отчета.
  • Ставка НДС – группировка используется в случае, если есть учет по разным ставкам НДС.
  • Учетная дата, месяц, квартал, год – указывает учетную дату начисления, т.е. когда эта услуга была оказана. Дата акта может отличаться от учетной даты услуги.

Стандартная настройка группировок включает в себя поля «Контрагент» и «Услуга». На основании такого отчета бухгалтер сможет ввести проводки по реализации услуг в базу бухгалтерии.

Колонки

  • Сумма – сумма услуг, проведенных по актам.
  • Количество – количество услуги, отраженное в актах.
  • Сумма НДС – сумма НДС по данным актов.
