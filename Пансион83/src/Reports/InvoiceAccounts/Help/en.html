Отчет "Взаиморасчеты по счетам-требованиям"

Назначение

Имя отчета в конфигураторе «InvoiceAccounts».

Отчет позволяет распечатать список счетов–требований для контрагентов с балансом по каждому счету и списком документов оплаты по счету.

Основное предназначение отчета - показывать на текущую дату список счетов по которым вовремя не поступила оплата. Отчет необходимо настроить по принятым в гостинице критериям не оплаты счета. Например, если счета должны оплачиваться не позднее 7 дней с даты выставления, то в отчете можно настроить отборы: "Возраст счета в днях" больше 7 и "Баланс по счету" больше 0.

Отборы на форме отчета

  • По гостинице - задает отбор по гостинице, если поле оставить пустым, то будут выведены данные по всем гостиницам.
  • По основным разрезам учета взаиморасчетов - по контрагенту, по договору.
  • По группе - задает отбор по группе гостей.
  • По фирме - задает отбор по фирме, если поле оставить пустым, то будут выведенные данные по всем фирмам.

Группировки

  • Фирма – юр.лицо гостиницы, от имени которой оформлено размещение гостя.
  • Договор – ссылка на договор из документа «Счет–требование».
  • Валюта – валюта документа «Счет–требование».
  • Контрагент – юр.лицо, заключившее с гостиницей договор, на основании которого произведено размещение гостя.
  • Количество оставшихся дней до заезда – количество дней, оставшихся от даты формирования отчета до дня заезда текущей группы.
  • Группа – объединяет несколько гостей, но может также состоять и из одного гостя. Каждый гость обязательно принадлежит какой–либо группе. Номер группы – это номер брони. В разрезе групп ведется учет взаиморасчетов с контрагентами.
  • Гостиница – название гостиницы, в которой размещен гость. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
  • Счет–требование – ссылка на документ «Счет-требование».
  • Регистратор – ссылка на документ «Счет-требование» или «Платеж»/«Платеж контрагента».

Колонки

  • Возраст счета в днях - количество дней, прошедших после даты выставления счета–требования.
  • Количество дней между датой платежа и датой заезда – вычисляется разность между датой планируемого заезда и датой оплаты счета. Используется в случае, когда условие оплаты счета формулируется как "оплата не позднее чем за 3 дня до даты планируемого заезда".
  • Задержка платежа в днях – количество дней между датой платежа и датой счета. Используется дляпоиска счетов по которым оплата пришла позже установленного числа дней. 
  • Баланс по счету – счет не оплачен или оплачен не полностью (+) или по счету получено денег больше чем требовалось (–).
  • Оплачено – сумма, оплаченная по счету.
  • Сумма счета – сумма, на которую выставлен счет-требование.
