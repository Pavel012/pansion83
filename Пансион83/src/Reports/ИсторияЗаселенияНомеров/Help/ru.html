Отчет "История заселения номеров"

Назначение

Имя отчета в конфигураторе «RoomsOccupationHistory».

Отчет выводит список гостей проживавших в номерах за указанный период по указанным гостиницам, номерам и типам номеров.

Отборы на форме отчета

  • По виду проверки периода отчета - проверка дат проживания относительно периода формирования отчета. Проверок существует насколько:
    • пересечение - в отчете отобразятся размещения, срок проживания которых пересекается с периодом отчета;
    • начинается в периоде - в отчет попадут только те размещения, дата заезда которых попадает в период отчета;
    • заканчивается в периоде - в отчет попадут только те размещения, дата выезда которых попадает в период отчета;
    • начинается или заканчивается в периоде - в отчет попадут только те размещения, дата заезда или выезда которых попадает в период отчета;
    • дата создания документа попадает в период - в отчете отобразятся размещения, которые были созданы в периоде отчета.
  • По номеру - задает отбор по номеру, если поле оставить пустым, то будут выведены данные по всем номерам гостиницы.
  • По типу номера - задает отбор по типу номера, если поле оставить пустым, то будут выведены данные по всем типам номеров гостиницы.
  • По проживающим или выселенным гостям. Флаг определяет показывать ли только проживающих гостей или и проживающих и выселенных.

Группировки

  • Статус размещения – разделитель учета, задается в документе «Размещение», обычно принимает значения «Заезд», «Выезд», «Переселение». Используется в отчетах и отборах.
  • Агент – ссылка на справочник контрагентов, указывает контрагента, который получает комиссию за гостя. Может также использоваться для аналитики работы агентов или туроператоров.
  • Автор – пользователь, создавший документ «Размещение».
  • Учетная дата заезда – дата заезда в текущем размещении.
  • День заезда - день заезда.
  • Час заезда - час заезда.
  • Месяц заезда - номер месяца заезда.
  • Квартал заезда - номер квартала заезда.
  • Неделя заезда - номер недели заезда.
  • Год заезда - год заезда.
  • Учетная дата выезда – дата выезда в текущем размещении.
  • Тип клиента – характеристика гостя, может использоваться как разделитель учета для аналитики, также используется для задания разных цен по разным типам клиентов (каждому типу клиента можно поставить в соответствие свой тариф).
  • Контактное лицо – ссылка на одноименное текстовое поле в документе «Бронь», указывает на лицо, бронировавшее данный номер от имени клиента. Обычно указывается при бронировании для контрагентов.
  • Договор – ссылка на договор контрагента, используется как разделитель учета взаиморасчетов с контрагентом.
  • Контрагент – ссылка на справочник контрагентов, указывает на организацию, юр. лицо, от имени которого было произведено бронирование (размещение).
  • Тип контрагента – характеристика контрагента. Может использоваться как разделитель учета для аналитики, также используется для задания разных цен по разным типам контрагентов (каждому типу контрагентов можно поставить в соответствие свой тариф).
  • Дисконтная карта – ссылка на справочник дисконтных карт. Заполняется при поселении, если гость предъявил карту.
  • Тип скидки – разделитель учета, ссылка на справочник «Тип скидок». Установка соответствия между набором услуг и размером скидки.
  • Запись в журнал регистрации иностранных граждан – ссылка на документ записи в журнал иностранцев (если текущий клиент иностранец).
  • Группа – объединяет несколько гостей, но может также состоять и из одного гостя. Каждый гость обязательно принадлежит какой–либо группе. Номер группы – это номер брони. В разрезе групп ведется учет взаиморасчетов с контрагентами.
  • Гостиница – название гостиницы, в которой размещен гость. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
  • Путевка/курсовка – ссылка на путевку, курсовка и пр.
  • Направление маркетинга – ссылка на элемент справочника «Направления маркетинга». Используется для анализа, может быть использован в группировках и отборах.
  • Планируемый способ оплаты – значение планируемого способа оплаты в документе «Размещение». Берется у первого фолио в таблице «Правила начислений». Обычно, если проживание оплачивает контрагент, то значение будет «На расчетный счет».
  • Цена – цена проживания в номере.
  • Регистратор – ссылка на документ «Размещение».
  • Номер – указывает номер комнаты.
  • Квота номеров – ссылка на справочник «Квоты номеров».
  • Тариф – показывает, по какому тарифу поселили гостя или забронировали номер. Тариф определяет цену номера и набор начисляемых услуг.
  • Тип тарифа – разделитель учета, в разрезе типа тарифа можно строить аналитические отчеты.
  • Тип номера – ссылка на справочник «Типы номеров», указывает на тип номера, в котором размещен гость (планируется размещение).
  • Источник информации – ссылка на справочник «Источник информации», может использоваться для построения аналитических отчетов.
  • Цель поездки – ссылка на справочник «Цели визита», может использоваться для построения аналитических отчетов.

Колонки

  • Вид размещения – ссылка на справочник «Виды размещений», указывает на вид текущего размещения.
  • Комиссия агента – сумма вычисленной комиссии для агента.
  • Тип комиссии агента.
  • Автомобиль.
  • Дата заезда – дата и время заезда в текущем размещении.
  • Дата выезда – дата и время выезда в текущем размещении.
  • Скидка – сумма предоставленной скидки.
  • Продолжительность.
  • Пустая колонка - колонка может быть преименованаи распечатат в отчете, например, если в списке нужна подпись гостей.
  • Гость – клиент, ссылка на справочник «Клиенты».
  • Валюта стоимости путевки - ссылка на справочник «Валюты».
  • Стоимость путевки - Указывает на стоимость путевки/курсовки.
  • Доп. мест – количество дополнительных мест в номере.
  • Мест – количество мест в номере.
  • Гостей – количество гостей в номере.
  • Номеров – количество номеров для текущего размещения.
  • Главный в группе – булев флаг, указание клиента, на которого начислены услуги группы.
  • Расхождение даты выезда с датой окончания действия миграционной карты - разница дат между окончанием действия миграционной карты и датой выселения иностранного гостя.
  • Количество мест в номере – для текущего номера показано количество мест согласно справочнику «Номерной фонд».
  • Количество человек в номере – для текущего номера показано количество человек согласно справочнику «Номерной фонд».
  • Документ основание – ссылка на документ «Размещение».
  • Примечания – текстовый комментарий.
  • Расхождение даты выезда с датой окончания действия визы - разница дат между окончанием действия визы и датой выселения иностранного гостя.
  • Продажи – сумма оказанных услуг за период от начала дня даты начала периода отчета до конца дня даты окончания периода отчета. Сумма продаж всегда выводится в отчетной валюте гостиницы.
  • Доходы от продажи номеров – сумма за проживание за период от начала дня даты начала периода отчета до конца дня даты окончания периода отчета. Доходы от продажи номеров всегда выводятся в отчетной валюте гостиницы.
  • Долг клиента – долг по лицевым счетам клиента (контрагент в лицевом счете пустой или равен "Физ. лица") на конец (23:59:59) даты окончания периода отчета. При расчете долги по лицевым счетам в разной валюте суммируются, поэтому эту колонку нельзя выводить в случае, когда у гостей могут быть лицевые счета в разных валютах.
