Отчет "Аудит событий системы безопасности"

Назначение

Имя отчета в конфигураторе «SafetySystemEventsAudit».

Отчет позволяет распечатать события регистрируемые системой безопасности в номерах. Есть возможность просматривать только события в пустных номерах, где не проживают гости.

Отборы на форме отчета

  • По типу события - задает отбор по типу события системы безопасности
  • По типу карты - задает отбор по типу карты, которой открывалась дверь или включалось электричество
  • По номеру.
  • По типу номера.
  • По гостинице.
  • Отбор только подозрительных событий. - задает отбор только тех событий, которые происходили в свободных не занятых номерах 

Группировки

  • Учетная дата - дата регистрации (без времени) события в программе.
  • Учетный час - час регистрации (без минут) события в программе.
  • Учетный месяц - месяц регистрации события в программе.
  • Учетный квартал - квартал регистрации события в программе.
  • Учетная неделя - неделя регистрации события в программе.
  • Учетный год - год регистрации события в программе.
  • Гостиница – название гостиницы, в которой зарегистрировано событие. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
  • Номер – указывает номер комнаты, в котором было зарегистрировано событие.
  • Тип номера - указывает категорию номера комнаты, в котором было зарегистрировано событие.
  • Тип события – тип зарегистрированного события.
  • Тип карты - тип карты, которая вызвала событие.
  • Автор события – пользователь действия которого привели к возникновению события.
  • Документ (бронь или размещение) по которому выдавалась карта-ключ
  • Гость, которому выдавалась карта ключ
  • Параметры фактического проживания гостей в номерах, которые пересекаются по времени с моментом регистрации события

Колонки

  • Период - дата и время регистрации события.
  • Гостиница – название гостиницы, в которой зарегистрировано событие. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
  • Номер – указывает номер комнаты, в котором было зарегистрировано событие.
  • Тип номера - указывает категорию номера комнаты, в котором было зарегистрировано событие.
  • Тип события – тип зарегистрированного события.
  • Тип карты - тип карты, которая вызвала событие.
  • Описание события – описание события как его вернула система безопасности.
  • Дата и время начала периода действия карты-ключа
  • Дата и время окончания периода действия карты-ключа
  • Документ (бронь или размещение) по которому выдавалась карта-ключ
  • Гость, которому выдавалась карта ключ
  • Кол-во событий зарегистрированных в выбранном периоде
  • Кол-во выданных карт-ключей
  • Параметры фактического проживания гостей в номерах
    • Документ – ссылка на документ «Размещение», который пересекается по времени с моментом регистрации события.
    • Гость - ссылка на гостя из размещения, которое пересекается по времени с моментом регистрации события
