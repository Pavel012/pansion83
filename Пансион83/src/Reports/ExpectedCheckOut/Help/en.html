Отчет "Планируемый выезд"

Назначение

Имя отчета в конфигураторе «ExpectedCheckOut».

Отчет выводит список проживающих гостей, которые должны выехать до указанной на форме отчета даты по указанным гостиницам, номерам и типам номеров.

Отборы на форме отчета

  • По гостинице - задает отбор по гостинице, если поле оставить пустым, то будут выведены данные по всем гостиницам.
  • По номеру - задает отбор по номеру. Если поле оставить пустым, то будут выведены данные по всем номерам.
  • По типу номера - задает отбор по типу номера». Если поле оставить пустым, то будут выведены данные по всем типам номеров.

Группировки

  • Статус размещения – элемент справочника «Статусы размещений», указывает на статус размещения данного клиента.
  • Агент – ссылка на справочник контрагентов, указывает контрагента, который получает комиссию за гостя. Может также использоваться для аналитики работы агентов или туроператоров.
  • Учетная дата заезда – дата заезда (без указания времени), указанная в документе «Бронь».
  • Учетная дата выезда – дата выезда (без указания времени), указанная в документе «Бронь».
  • День выезда – день недели выезда гостя. Может использоваться для группировки и отбора.
  • Час выезда – час, когда гость выехал. Может использоваться для группировки и отбора, например, можно получить отчет который покажет распределение количества заехавших гостей по часам.
  • Месяц выезда – месяц выезда гостя, используется для группировки и отборов.
  • Квартал выезда – квартал даты выезда, для группировки и отборов.
  • Неделя выезда – номер недели выезда гостя.
  • Год выезда – значение года выезда гостя.
  • Тип клиента – характеристика гостя, может использоваться как разделитель учета для аналитики, также используется для задания разных цен по разным типам клиентов (каждому типу клиента можно поставить в соответствие свой тариф).
  • Контактное лицо – ссылка на одноименное текстовое поле в документе «Бронь», указывает на лицо, бронировавшее данный номер от имени клиента. Обычно указывается при бронировании для контрагентов.
  • Договор – ссылка на договор контрагента, используется как разделитель учета взаиморасчетов с контрагентом.
  • Контрагент – юр.лицо, заключившее с гостиницей договор, на основании которого произведено размещение гостя.
  • Тип контрагента – характеристика контрагента, может использоваться как разделитель учета для аналитики, также используется для задания разных цен по разным типам контрагентов (каждому типу контрагентов можно поставить в соответствие свой тариф).
  • Дисконтная карта – ссылка на справочник дисконтных карт. Заполняется при поселении, если гость предъявил карту.
  • Тип скидки – разделитель учета, ссылка на справочник «Тип скидок». Установка соответствия между набором услуг и размером скидки.
  • Группа – объединяет несколько гостей, но может также состоять и из одного гостя. Каждый гость обязательно принадлежит какой–либо группе.
  • Гостиница – название гостиницы, в которой размещен гость. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
  • Путевка – путевки, курсовки и пр.
  • Направление маркетинга – ссылка на элемент справочника «Направления маркетинга». Используется для анализа, может быть использован в группировках и отборах.
  • Планируемый способ оплаты – ссылка на справочник «Способы оплаты». Разделитель учета, позволяющий указать, каким образом осуществляется оплата текущего гостя.
  • Цена – цена проживания.
  • Регистратор – ссылка на документ «Бронь».
  • Номер – номер комнаты, из которой планируется выезд.
  • Квота номеров – ссылка на справочник «Квоты номеров».
  • Тариф – показывает, по какому тарифу поселили гостя, или забронировали номер. Тариф определяет цену номера и набор начисляемых услуг.
  • Тип тарифа – разделитель учета, в разрезе типа тарифа можно строить аналитические отчеты.
  • Тип номера – ссылка на справочник «Типы номеров», указывает на тип номера, в котором размещен гость.
  • Источник информации – ссылка на справочник «Источник информации», может использоваться для построения аналитических отчетов.
  • Цель поездки – ссылка на справочник «Цели визита», может использоваться для построения аналитических отчетов.

Колонки

  • Вид размещения – элемент справочника «Виды размещений», указывает на вид размещения данного клиента.
  • Комиссия агента – сумма вычисленной комиссии для агента.
  • Тип комиссии агента.
  • Автомобиль – ссылка на поле документа «Бронь».
  • Дата заезда – дата и время заезда, указанная в документе «Бронь».
  • Дата выезда – дата и время выезда, указанная в документе «Бронь».
  • Баланс клиента – значение долга гостя.
  • Скидка – сумма предоставленной скидки.
  • Продолжительность.
  • Гость – клиент, справочник «Клиенты».
  • Доп. мест – количество дополнительных мест, забронированных в данной группе.
  • Мест - количество мест, забронированных в данной группе.
  • Гостей – количество гостей в группе.
  • Номеров – количество номеров, планируемых к выселению в данной группе.
  • Главный в группе – булев флаг, указывающий для нескольких Клиентов в одной группе того, который оплачивает лицевые счета.
  • Кол-во мест в номере – общее количество мест в номере, в который поселили гостя. Не зависит от размещения, зависит только от номера, в который гостя поселили.
  • Кол-во человек в номере – свойство номера, в который поселили гостя, отражает общее максимальное количество гостей, которое можно поселить в выбранный номер. Не зависит о размещения, зависит только от номера.
  • Документ основание – ссылка на документ «Бронь».
  • Момент времени – содержит позицию документа «Размещения», которая показывает точное время оформления гостя в базе данных.
  • Примечания – текстовый комментарий.
