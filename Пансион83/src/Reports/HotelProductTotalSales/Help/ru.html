Отчет "Полные продажи путевок"

Назначение

Имя отчета в конфигураторе «HotelProductTotalSales».

Отчет используется для формирования различных реестров по проданным путевкам и курсовкам. Отчет по каждой путевке показывает ее полную стоимость. В отчет попадают только те путевки/курсовки, у которых дата продажи (которая во многих случаях совпадает с датой заезда) попадает в период отчета. В колонке "Стоимость путевки" отчет всегда отражает полную сумму оказанных услуг по путевке, не разбивая ее по учетным датам услуг проживания и питания и не обрезая ее по периоду отчета.

Отчет может использоваться для проведения в бухгалтерии проводок по счету 98 - "Доходы будущих периодов".

Отчет позволяет по каждой путевке показать сумму закрытую актами (включатся флажком "Показывать суммы по путевкам закрытые актами") и сумму оплаты (включается флажками ""Показывать платежи по группам гостей" и "Разделять суммы платежей по колонкам типов оплаты"). Если флажок "Разделять суммы платежей по колонкам типов оплаты" выключен, то программа заполняет колонку отчета "Способ оплаты".

Отчет позволяет показать по каждой путевке ту сумму из стоимости путевки, которая приходится на период отчета. Эти данные находятся в колонке "Продажи за период" и рассчитываются при включенном флажке "Показывать текущие продажи по путевкам".

Отборы на форме отчета

  • По гостинице - задает отбор по гостинице, если поле оставить пустым, то будут выведены данные по всем гостиницам.
  • По путевке - задает отбор по типу путевок/курсовок или по конкретной путевке/курсовке.
  • По основным разрезам учета взаиморасчетов - по контрагенту, по договору, по группе гостей.
  • По набору услуг - задает отбор по набору услуг путевки.

Группировки

  • Вид размещения – справочник Видов размещения.
  • Агент – ссылка на справочник контрагентов, указывает контрагента, который получает комиссию за гостя. Может также использоваться для аналитики работы агентов или туроператоров.
  • Клиент – справочник «Клиенты», гость в данном размещении.
  • Тип клиента – для указания «бесплатных» гостей.
  • Договор – договор из фолио, по которому проведено начисление.
  • Контрагент – юр.лицо, заключившее с гостиницей договор, на основании которого произведено размещение гостя.
  • Тип скидки – разделитель учета, ссылка на справочник «Тип скидок». Установка соответствия между набором услуг и размером скидки.
  • Продолжительность.
  • Дата оказания - дата продажи путевки (обычно совпадает с датой заезда).
  • Группа – объединяет несколько гостей, но может также состоять и из одного гостя. Каждый гость обязательно принадлежит какой–либо группе. Номер группы – это номер брони. В разрезе групп ведется учет взаиморасчетов с контрагентами.
  • Гостиница – название гостиницы, в которой размещен гость. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
  • Путевка – номер бланка путевки или курсовки.
  • Тип путевки - группа справочника путевок и курсовок.
  • Направление маркетинга – ссылка на элемент справочника «Направления маркетинга». Используется для анализа, может быть использован в группировках и отборах.
  • Гостей – число гостей в данной группе.
  • Планируемый способ оплаты – ссылка на справочник «Способы оплаты». Разделитель учета, позволяющий указать, каким образом осуществляется оплата текущего гостя.
  • Отчетная валюта – валюта, обозначенная как отчетная в карточке Гостиницы.
  • Квота номеров – квота номеров, соответствующая данной путевке или курсовке.
  • Тариф – показывает, по какому тарифу поселили гостя, или забронировали номер. Тариф определяет цену номера и набор начисляемых услуг Доходы от продажи номеров– сумма только от проживания гостя, без учета дополнительных услуг. Скидки учтены.
  • Тип номера – ссылка на справочник «Типы номеров», указывает на тип номера, в котором размещен гость(планируется размещение).
  • Источник информации – ссылка на справочник «Источник информации», может использоваться для построения аналитических отчетов.
  • Цель поездки – ссылка на справочник «Цели визита», может использоваться для построения аналитических отчетов.

Колонки

  • Продано дополнительных мест – количество проданых дополнительных мест.
  • Продано мест – количество мест, проданных в данной группе.
  • Дата заезда – дата заезда данной группы гостей.
  • Дата выезда – дата выезда данной группы гостей.
  • Комиссия – сумма начисленной комиссии.
  • Комиссия без НДС – та же сумма за вычетом НДС.
  • Количество путевок – количество проданных в данной группе путевок.
  • Сумма скидки – сумма скидки по отношению к тарифу размещения.
  • Сумма скидки без НДС.
  • Человеко-дни – количество человеко-дней. Рассчитывается по каждому гостю независимо от стоимости проживания.
  • Заезд гостей – количество заехавших гостей за период отчета.
  • Доходы от продажи номеров - сумма только от проживания гостя, без учета дополнительных услуг. Скидки учтены.
  • Доходы от продажи номеров без НДС – предыдущая сумма без НДС.
  • Продано номеров – количество проданных в данной группе номеров.
  • Стоимость путевки – общая сумма продаж услуг по путевке/курсовке.
  • Цена путевки – стоимость путевки указанная в карточке путевке (плановая стоимость).
  • Продажи за период – сумма продаж услуг по путевке/курсовке строго за период отчета
  • Сумма в актах – сумма продаж услуг по путевке/курсовке на которую сформированы акты об оказании услуг
  • Сумма в актах в периоде отчета – сумма продаж услуг по путевке/курсовке закрытая актами, которые созданы в периоде отчета
  • Сумма оплаты – сумма всех платежей по лицевым счетам где встречаются услуги начисленные по путевке. Если в параметрах отчета указана секция платежей по путевкам, то отбираются суммы платежей по этой секции.
  • Оплата наличными – сумма всех наличных платежей по лицевым счетам где встречаются услуги начисленные по путевке. При добавлении в отчет колонок разделения оплаченных сумм по видам оплаты желательно включить в параметрах отчета флажок "Разделять суммы платежей по колонкам типов оплаты". В этом случае программа не будет заполнять колонку "Способ оплаты путевки/курсовки", т.к. в этом случае допускается смешанная оплата путевок.
  • Оплата кред. картой – сумма всех платежей по кредитным картам по лицевым счетам где встречаются услуги начисленные по путевке.
  • Оплата б/н – сумма всех банковских переводов на расчетный счет гостиницы по лицевым счетам где встречаются услуги начисленные по путевке.
  • Ресурсы без НДС – в РФ суммы совпадают с суммами включающими НДС.
