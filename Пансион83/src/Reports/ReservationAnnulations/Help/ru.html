Отчет "Аудит аннуляций брони"

Назначение

Имя отчета в конфигураторе «ReservationAnnulations».

Отчет используется для аудита аннулированной (отмененной) брони. Отчет выводит бронь аннулированную за указанный период. Бронь считается аннулированной если у нее установлен не действующий статус и бронь не находится в листе ожидания и не является предварительной. Бронь в статусе "Незаезд" так же считается аннулированной. 

Отборы на форме отчета

  • Период отчета – отбирает брони аннулированные в выбранном периоде (по дате и времени установки не действующего статуса брони).
  • Гостиница – задает отбор по гостинице, если поле оставить пустым, то будут показаны данные по всем гостиницам.
  • Номер – задает отбор по номеру комнаты или по группе номеров.
  • Тип номера – задает отбор по типу номера или по группе типов номеров.
  • Сотрудник – задает отбор по сотруднику, проведшему аннулирование.

    Группировки

    • Статус брони – разделитель учета, задается в документе «Бронь», обычно принимает значения «Действует», «Заезд», «Незаезд», «Отказ» и т.д. Используется в отчетах и отборах.
    • Вид размещения — ссылка на справочник «Виды размещений», указывает на вид текущего размещения.
    • Агент – ссылка на справочник контрагентов, указывает контрагента, который получает комиссию за гостя. Может также использоваться для аналитики работы агентов или туроператоров.
    • Автор – сотрудник, оформивший гостя (пользователь, создавший документ «Размещение»).
    • Учетная дата аннуляции – дата аннуляции брони. Не содержит времени аннуляции. Используется для отбора и группировки.
    • Учетная дата заезда – дата, планируемого заезда гостя. Не содержит времени заезда. Используется для отбора и группировки.
    • День заезда – день недели планируемого заезда гостя. Может использоваться для группировки и отбора.
    • Час заезда – час планируемого заезда гостя. Может использоваться для группировки и отбора, например, можно получить отчет который покажет распределение количества заезжающих гостей по часам.
    • Месяц заезда – месяц планируемого заезда гостя, используется для группировки и отборов.
    • Квартал заезда – квартал даты планируемого заезда гостя, используется для группировки и отборов.
    • Неделя заезда – номер недели от даты планируемого заезда гостя.
    • Год заезда – значение года планируемого заезда гостя.
    • Учетная дата выезда – дата, когда гость планировал выехать. Не учитывается время. Используется для отбора и группировки.
    • Тип клиента – значение типа клиента. Ссылка на справочник «Типы клиентов».
    • Контактное лицо – ссылка на одноименное текстовое поле в документе «Бронь». Указывает на лицо, бронировавшее данный номер от имени клиента. Обычно указывается при бронировании для контрагентов.
    • Договор – значение договора из «Брони».
    • Контрагент – ссылка на справочник «Контрагенты», от имени которого приехал гость. От имени этого контрагента обычно и было оформлено бронирование.
    • Тип контрагента – значение типа контрагента, указанное в «Брони». Может использоваться для группировки и отбора.
    • Дисконтная карта – ссылка на справочник дисконтных карт. Заполняется при поселении или бронировании, если гость предъявил карту или продиктовал ее номер.
    • Тип скидки – разделитель учета, ссылка на справочник «Тип скидок». Установка соответствия между набором услуг и размером скидки.
    • Гость – ссылка на справочник «Клиентов», карточка гостя, которого планируется поселить.
    • Группа – объединяет несколько гостей, но может также состоять и из одного гостя. Каждый гость обязательно принадлежит какой-либо группе. Номер группы – это номер брони. В разрезе групп ведется учет взаиморасчетов с контрагентами.
    • Гостиница – название гостиницы, в которой размещен гость. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
    • Путевка или курсовка – ссылка на номер путевки/курсовки по которой поселили гостя.
    • Направление маркетинга – ссылка на элемент справочника «Направления маркетинга». Используется для анализа, может быть использован в группировках и отборах.
    • Планируемый способ оплаты – значение планируемого способа оплаты в документе «Бронь». Берется у первого фолио в таблице «Правила начислений». Обычно, если проживание оплачивает контрагент, то значение будет «На расчетный счет».
    • Цена – значение цены за сутки проживания (или за час, если тариф почасовой). Строка, включающая знак валюты и тип дня календаря. Значение заполняется автоматически при проведении «Брони».
    • Регистратор – ссылка на документ «Бронь» которым был оформлен гость.
    • Номер – номер забронированной комнаты.
    • Квота номеров – значение квоты номеров, по которой была оформлена бронь.
    • Тариф – показывает, по какому тарифу бронировали гостя. Тариф определяет цену номера и набор начисляемых услуг.
    • Тип тарифа – разделитель учета, в разрезе типа тарифа можно строить аналитические отчеты.
    • Тип номера – ссылка на справочник «Типы номеров», указывает на тип номера, в котором планируется размещение гостя.
    • Источник информации – ссылка на аналитический справочник «Источник информации о гостинице». Используется для анализа рекламных источников. Данное значение может быть обязательным полем для заполнения при бронировании.
    • Цель поездки – ссылка на справочник «Цели поездки». Используется для анализа. Может использоваться в группировках и отборах. Данное значение в «Брони» может быть обязательным для заполнения полем.

    Колонки

    • Доп. мест – количество дополнительных мест, которые занимает гость.
    • Комиссия агента – значение комиссии агента, начисляемой за проживание гостя.
    • Тип комиссии агента – значение типа комиссии, может быть одно из значений перечисления «Виды комиссий агентам». Стандартное значение – «%», процент.
    • Автор аннуляции - сотрудник, проведший аннуляцию брони.
    • Мест – количество мест в номере, занятых гостем.
    • Автомобиль – строковое значение поля «Автомобиль» из размещения, которое заполняется вручную при поселении или при бронировании.
    • Дата заезда – дата и время заезда гостя.
    • Дата выезда – значение даты и времени выезда.
    • Скидка – значение скидки в процентах.
    • Продолжительность – указывается в днях или часах, в зависимости от выбранного тарифа. Если в тарифе цена указана за сутки, то продолжительность рассчитывается в днях. Если цена указывается за час, то продолжительность рассчитывается в часах.
    • Гостей – количество гостей оформленных документом «Бронь».
    • Главный в группе – булево значение флажка отмечающего главного в группе, на которого будут произведены все начисления по группе. Для корректного отражения в отчете используйте «Формат» в таблице колонок.
    • Кол-во мест в номере – общее количество мест в номере, в который поселили гостя. Не зависит от размещения, зависит только от номера, в который гостя поселили.
    • Кол-во человек в номере – свойство номера, в который поселили гостя, отражает общее максимальное количество гостей, которое можно поселить в выбранный номер. Не зависит о размещения, зависит только от номера.
    • Цена в брони – по какой цене было произведено бронирование.
    • Количество аннулирований - количество аннулированных документов "Бронь".
    • Примечания – срока, значение из документа «Бронь».
    • Номеров – количество номеров занятых данным гостем.
