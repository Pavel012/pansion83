Отчет "Аудит аннуляций платежей"

Назначение

Имя отчета в конфигураторе «PaymentAnnulations».

Отчет позволяет получить распечатку аннулированных платежей за указанный период. Отчет позволяет по каждому сотруднику посчитать количество аннуляций за период и может использоваться для аудита аннулированных платежей.

Отборы на форме отчета

  • По гостинице - задает отбор по гостинице, если поле оставить пустым, то будут выведены данные по всем гостиницам.
  • По сотруднику - задает отбор по сотруднику. Если поле оставить пустым, то будут выведенные сотрудникам.
  • По кассовой секции - учитывает определенную кассовую секцию.
  • По ККМ.
  • По способу оплаты - отбирает записи только по выбранному способу оплаты. Если значение поля пустое, то в отчете отобразятся данные по всем способам оплаты.

Группировки

  • Учетная дата - дата учета аннуляции.
  • Учетный месяц - номер месяца учета.
  • Учетный квартал - номер квартала учета аннуляции.
  • Учетная неделя - номер недели.
  • Учетный год - год аннуляции.
  • Автор платежа - сотрудник гостиницы, который провел платеж.
  • Автор аннуляции - сотрудник гостиницы, который провел аннуляцию платежа.
  • ККМ - контрольно-кассовый аппарат, по которому проведен платеж и аннуляция.
  • Фирма - юр.лицо гостиницы, от имени которой оформлено размещение гостя.
  • Гостиница – название гостиницы, в которой размещен гость. Если в одной базе ведется учет по нескольким гостиницам, используется как разделитель учета.
  • Документ основание – документ, который послужил основанием для оплаты услуги.
  • Плательщик - гость или контрагент, оплативший счет.
  • Платеж - ссылка на документ «Платеж».
  • Валюта платежа - валюта, в которой был проведен платеж.
  • Способ оплаты - способ облаты счета.
  • Кассовая секция - секция ККМ. Программа позволяет проводить платежи по различным кассовым секциям.
  • Тип аннуляции - принимает одно из 2 возможных значений: «Отказ клиента» или «Ошибка сотрудника».
  • Ставка НДС - значение процента налога на добавленную стоимость.

Колонки

  • Дата создания платежа - дата и время проведения платежа.
  • Дата аннуляции платежа - дата и всеря проведения ануляциии.
  • Количество аннулирований - принимает в строке значение «1». Необходимо поле для вывода итогового значения.
  • Примечания - текстовое поле, заполняемое при аннуляции платежа вместе с типом аннуляции.
  • Сумма - сумма аннуляции. По умолчанию - это сумма аннулируемого платежа.
  • Сумма НДС - сумма по ставке НДС.
