Документ "Приказ об изменении цен тарифа"

Назначение

В приказах об изменении цен тарифа указывается состав и цены услуг, которые будут автоматически начисляться гостям при бронировании и проживании. Дата и время документа задают момент времени, когда приказ начинает действовать. Программа определяет, какие цены применить к периоду проживания гостя, путем поиска действующего на дату и время заезда гостя приказа об изменении цен.

Цены всегда задаются для какого-либо тарифа и типа дня календаря. Эти параметры обязательны для ввода.

Цены в приказе задаются в разрезе типов клиентов. На форме приказа всегда можно указать цены для пустого типа клиента. Цены по пустому типу клиента используются программой в случаях, когда в брони/размещении тип клиента не указан или указан, но для него не определены цены в приказах.

Относится к подсистеме: «Управление взаиморасчетами».

Параметры

Длина номера: 12, тип номера: Строка

Периодичность номера: Год

Тип номера: Строка

Вводится на основании: «Тарифы».

Реквизиты

  • Гостиница. Ссылается на справочник «Гостиницы».
  • Тариф. Ссылается на справочник «Тарифы». Обязательное для заполнения поле. Тариф по которому вводится приказ.
  • Тип дня календаря. Ссылается на справочник «Типы дней в календарях». Обязательное для заполнения поле. Тип дня календаря для которого в приказе будут указаны цены.
  • Примечания. Тип значения: «Строка». Произвольные примечания к документу.
  • Автор. Ссылается на справочник «Сотрудники». Сотрудник, который создал документ.

Табличная часть «Цены»

  • Тип клиента. Ссылается на справочник «Типы клиентов». Поле непосредственно в таблице не редактируется, а отображается в виде закладок над таблицей услуг. Закладки выводятся по всем не помеченным на удаление типам клиентов из справочника.
  • Тип номера. Ссылается на справочник «Типы номеров». Тип номера по которому будет начисляться услуга. Можно указать папку типов номеров или оставить пустым. Если оставить пустым, то при проведении документа в регистр цен услуг по тарифам будут добавлены строки для всех не помеченных на удаление типов номеров из справочника. Если указана папка типов номеров, то при проведении приказа в регистр цен услуг по тарифам будут добавлены строки для всех не помеченных на удаление типов номеров из выбранной папки
  • Вид размещения. Ссылается на справочник «Виды размещения». Вид размещения по которому будет начисляться услуга. Можно указать папку видов размещений или оставить пустым. Если оставить пустым, то при проведении документа в регистр цен услуг по тарифам будут добавлены строки для всех не помеченных на удаление видов размещений из справочника. Если указана папка видов размещений, то при проведении приказа в регистр цен услуг по тарифам будут добавлены строки для всех не помеченных на удаление видов размещений из выбранной папки
  • Услуга. Ссылается на справочник «Услуги». Услуга, которая будет начисляться
  • Цена. Тип значения: «Число». Цена услуги
  • Валюта. Ссылается на справочник «Валюты». Валюта цены услуги
  • Минимальное количество. Тип значения: «Число». Минимально возможное к начислению количество. Заполняется обычно в почасовых тарифах в случаях, когда проживание в номере не может оплачиваться меньше чем за, например, 2 часа.
  • Ставка НДС %. Ссылается на справочник «Ставки НДС». Ставка НДС услуги.
  • Правило вычисления количества. Ссылается на справочник «Правила вычисления количества услуг». Правило по которому программа будет рассчитывать в брони и размещении количество оказания услуги. Поле должно быть заполнено. Может ссылаться на внешний пользовательский алгоритм.
  • Это доход от продажи номеров. Тип значения: «Булево». Флажок включается обычно у услуг проживания и означает, что по этой услуге программа будет рассчитывать число проданных номеров/мест, заезд гостей, доход от продажи номеров
  • Входит в цену. Тип значения: «Булево». Флажок включается обычно у услуг проживания и питания. Если флажок включен и у услуг питания явно указана стоимость, то на форме, например, "Что свободно" в колонке "Цена" будет показана стоимость проживания за сутки включая стоимость питания. если в приказе флажок "Входит в цену" у услуг питания выключен, то на форме, например, "Что свободно" в колонке "Цена" будет показана стоимость за сутки только по услуге проживания.
  • Это цена за одного человека. Тип значения: «Булево». Если флажок включен, то при начислении услуги в размещении/брони, количество услуги рассчитанное по периоду проживания и по указанному в приказе алгоритму будет умножаться на число гостей указанное в брони/размещении.

Формы

  • Форма документа. Основная форма объекта. Используется для заполнения обязательных реквизитов.
  • Форма списка. Основная форма журнала приказов. Используется для отображения списка или выбора документов данного типа.

Способ вызова

Вызывается из меню "Взаиморасчеты/Справочники/Приказ об изменени цен" или из главного меню программы по кнопке "Еще../Закладка "2"/Тарифы", кнопка "Перейти" в панели инструментов списка тарифов. Также приказы редактируются в карточке тарифа на закладке "Приказы".

Движения

  • «Цены по тарифам».
  • «Действующие приказы по тарифам».

Дополнительная информация

Если начисления по раннему заезду и позднему выезду рассчитываются по внешнему алгоритму, то они должны быть определены как услуги и тоже должны входить в состав услуг, перечисляемых в приказе. Если гостиница берет деньги за бронирование (например 25% от стоимости первых суток), то услуга "Бронирование 25%" также должна быть указана в приказе об изменении цен со своим специальным алгоритмом расчета количества.
