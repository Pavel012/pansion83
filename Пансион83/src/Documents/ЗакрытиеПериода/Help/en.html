Документ "Закрытие периода"

Назначение

Документ используется для сбора реализации услуг по фирмам и контрагентам для целей бухгалтерского учета и учета взаиморасчетов с контрагентами. Закрытие периода можно производить вручную или автоматически с помощью регламентной процедуры. Настройка автоматического закрытия периода осуществляется в справочнике «Фирмы» на закладке «Учетная политика».

При закрытии периода формируются документы "Акт", которые выгружаются в бухгалтерию. Рекомендуется настроить автоматическое закрытие периода в справочнике "Фирмы". Рекомендуемое время закрытия 03:00 ежедневно.

Транзакции собираются по регистру "Текущая реализация услуг". Каждая транзакция (документ "Начисление") делает движение вида "Приход" в этом регистре. Когда вводится акт, то по регистру делается движение вида "Расход". Таким образом баланс по каждой транзакции стремится к нулю, что позволяет автоматически выявлять и подхватывать изменения в сумме или аналитике начислений.

Логика формирования актов задается в справочнике "Фирмы" на закладке "Учетная политика".  

При необходимости переформировать акты по уже закрытому периоду перед перепроведением документа необходимо пометить на удаление все акты созданные этим закрытием. Если акты не удалять, то при перепроведения периода новые акты созданы не будут!

Относится к подсистеме: «Управление взаиморасчетами».

Параметры

Длина номера: 12, тип номера: Строка

Периодичность номера: Год

Тип номера: Строка

Вводится на основании: «Фирмы».

Реквизиты

  • Гостиница. Ссылается на справочник «Гостиницы». Указывается гостиница, реализацию по которой необходимо собрать. Если не указано, то будут собраны транзакции по всем гостиницам.
  • Фирма. Ссылается на справочник «Фирмы». Указывается по какой фирме отбирать начисления. Акты будут сформированы на указанную фирму.
  • Обрабатывать только закрытые лицевые счета. Если флажок включен, то в акты попадут только транзакции по лицевым счетам со статусом "Закрыт", т.е. по счетам выехавших гостей.
  • Создавать один акт для всех групп гостей от контрагента/договора. Если флажок включен, то все начисления одного контргаента будут собраны в один акт. Если флажок выключен, то программа будет создавать акты отдельнона каждую группу гостей.
  • Создавать один акт для всех групп гостей - частных лиц. Если включен, то начисления по лицевым счетам гостей, кторые оплачены на стойке наличными, будут собраны в один акт. Если выключен, то акты будут формироваться в соответсвии с учетной политикой, установленной в справочнике "Фирмы". 
  • Закрывать фактическое оказание услуг. Если флажок включен, то при проведении документа, услуги запланированные к оказанию в регистре «Регистрация услуг» и у которых дата оказания меньше или равна дате закрытия периода, будут списаны. Используется в тех случаях, когда в гостинице принято,что не использованные услуги питания нельзя переносить с прошлых дней на текущий день.
  • Примечания. Комментарий к документу.
  • Автор. Ссылается на справочник «Сотрудники». Пользователь, который ввел операцию.

Способ вызова

Для создания документа ручного закрытия периода необходимо нажать на кнопку «Добавить» в панели инструментов журнала закрытий периодов, который открывается из главного меню программы по кнопке "Еще.../Закрытия периодов".
