Обработка "Очистить базу данных"

Назначение

Обработка используется для очистки базы данных. Обработка выполняет следующие действия:

  1. Удаляет из базы данных помеченные на удаление начисления, на которые нет ссылок.
  2. Удаляет из базы данных помеченные на удаление лицевые счета на которые нет ссылок.
  3. Удаляет записи из регистров хранения истории изменения документов, клиентов и контрагентов.
  4. Помечает на удаление карточки клиентов, на которые нет ссылок в брони, размещениях, брони ресурсов и лицевых счетах и у которых не заполнены паспортные данные. Эта процедура убирает из выдачи формы поиска подходящего клиента карточки, по которым нет истории в базе данных
  5. Удаляет сканы документов удостоверяющих личность клиентов.
    Если флажок "Сохранять обработанные сканы документов в папку на диске" включен, то документы не удаляются, а картинки из документов сохраняются в папку "Корневая папка для хранения BLOB данных", указанную в параметрах отеля. При этом сохраняется возможность просматривать и распознавать документы. Если флажок "Сохранять обработанные сканы документов в папку на диске" выключен, то документы удаляются из базы данных полностью, без возможности их восстановления и просмотра.
    Если флажок "Удалять только обработанные сканы документов" включен, то удаляются только проведенные документы в статусе "Обработан". Иначе удаляются все документы за указанный период.
  6. Из карточек клиентов очищает идентификационные данные: паспортные данные, очищает в адресе проживания улицу, дом, квартиру, оставляя только регион, область и город. Включать эту галочку нужно только в случае необходимости проведения обезличивания карточек клиентов по требованию контролирующих органов.
  7. Удаляет программные сообщения.
  8. Удаляет отправленные СМС сообщения и документы рассылки СМС сообщений.

Удаляются только объекты и записи, дата которых не позже указанной на форме обработки даты.

Относится к подсистеме: «Управление настройками».

Способ вызова

Вызывается из меню "Обработки"

Дополнительная информация

Рекомендуемый режим выполнения обработки: ежедневное фоновое задание, время выполнения 04:00

При необходимости выполнять очистку данных с разной глубиной истории, например, удалять сканы данных клиентов страше 1 месяца, а очищать идентификационные данные старше 1 года, рекомендуется создать в справочнике обработок копию обработки со своей настройкой параметров и своим расписанием фонового выполнения.
